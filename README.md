# MetaEngine

MetaEngine сейчас находится на этапе **локальной лаборатории для проверки стратегий, пресетов и базовых расчетов**.

Цель текущей версии — быстро проверять CSV-файлы, собирать из них пресеты с весами и периодами, считать базовые ряды и смотреть результат в браузере.

> Важно: это пока **файловый прототип** на Node.js для проверки логики. По ТЗ production-версия позже должна быть на ASP.NET Core / C# / PostgreSQL.

## Что уже умеет локальная версия

- Запуск локальной страницы в браузере.
- Загрузка CSV-стратегий без заголовков в формате `timestamp,value`.
- Выбор типа значения при загрузке:
  - `diff` — доходность за шаг;
  - `accum` — накопленная доходность.
- Нормализация загруженной стратегии в формат `timestamp,diff`.
- Пересчет `accum → diff`:
  - первая точка получает `diff = 0`;
  - пропуски в `accum` заполняются предыдущим известным значением;
  - после пересчета сохраняется только `diff`.
- Хранение стратегий в `samples/strategies`.
- Удаление стратегий через интерфейс.
- Создание пресетов с:
  - названием;
  - стратегиями;
  - весами в процентах;
  - периодами участия;
  - режимом «До конца» для открытого периода.
- Поддержка ребалансировок: одну стратегию можно добавить в пресет несколько раз с разными непересекающимися периодами.
- Хранение пресетов в `samples/presets`.
- Удаление пресетов через интерфейс и API.
- Расчет одной стратегии или пресета за выбранный период.
- Расчет базовых рядов:
  - `diff`;
  - `accum`;
  - `hwm`;
  - `dd`;
  - `mdd`.
- Отображение результата:
  - итоговой таблицей;
  - таблицей по точкам;
  - графиком с тумблерами линий.
- Цветные тумблеры графика: цвет тумблера совпадает с цветом линии.
- Date picker для выбора периодов.
- Предупреждения о пропусках в данных.
- Сохранение результатов расчетов в `samples/runs`.

## Формат CSV стратегии

Ожидается CSV без заголовков:

```csv
1734606000000,0
1734609600000,0.0001422356863014
1734613200000,-0.0005010288408471
```

То есть:

```text
timestamp,value
```

Где:

- первая колонка — timestamp;
- вторая колонка — `diff` или `accum`;
- тип второй колонки выбирается в интерфейсе при загрузке.

После загрузки стратегия сохраняется в `samples/strategies` уже в нормализованном формате:

```csv
timestamp,diff
1734606000000,0
1734609600000,0.0001422356863014
```

Если был загружен `accum`, он пересчитывается в `diff`, и дальше система работает уже через `diff`.

## Время и даты

В интерфейсе даты показываются в формате:

```text
YYYY-MM-DD HH:MM
```

Примеры:

```text
2026-04-30 14:00
2025-04-09 23:00
2021-10-03 23:00
```

Timestamp читается как Unix timestamp в миллисекундах. Внутри используется UTC-форматирование без дополнительного пользовательского сдвига часового пояса.

## Правило отсутствующих данных

Единое правило:

```text
если данных нет для diff → берем 0
если данных нет для accum → берем предыдущее известное значение
```

На практике после загрузки все стратегии хранятся как `diff`, поэтому при расчетах отсутствующие значения дают `diff = 0`.

Пример для `diff`:

```csv
timestamp,diff
2024-01-06 00:00,0
2024-01-06 01:00,-0.0003
2024-01-06 03:00,0.0002
```

Для расчета это будет обработано так:

```csv
timestamp,diff
2024-01-06 00:00,0
2024-01-06 01:00,-0.0003
2024-01-06 02:00,0
2024-01-06 03:00,0.0002
```

Пример для `accum`:

```csv
timestamp,accum
2024-01-06 00:00,0.1
2024-01-06 01:00,0.2
2024-01-06 03:00,0.3
```

Логически пропуск заполняется так:

```csv
timestamp,accum
2024-01-06 00:00,0.1
2024-01-06 01:00,0.2
2024-01-06 02:00,0.2
2024-01-06 03:00,0.3
```

Потом `accum` пересчитывается в `diff`.

## Стратегии

Стратегия — это CSV-файл, сохраненный в:

```text
samples/strategies
```

Через интерфейс можно:

- загрузить стратегию;
- выбрать, во второй колонке `diff` или `accum`;
- посмотреть период данных, шаг и количество пропусков;
- удалить стратегию.

Если стратегия используется в пресете и ее удалить:

- пресет не удаляется;
- при расчете по этой стратегии будут нули;
- если позже загрузить стратегию с тем же именем, она снова начнет участвовать в расчете пресета.

## Пресеты

Пресет — это рецепт из стратегий, весов и периодов.

Пример:

```text
preset_1
- strategy_a: вес 100%, период 2024-01-01 00:00 — 2025-01-01 00:00
- strategy_a: вес 10%, период 2025-01-01 00:00 — до конца
- strategy_b: вес 25%, период 2024-06-01 00:00 — до конца
```

Веса пользователь вводит в процентах:

```text
100 = 100%
25 = 25%
-20 = -20%
150 = 150%
```

Внутри расчета веса применяются как доли:

```text
100% → 1.0
25% → 0.25
-20% → -0.2
150% → 1.5
```

Пресеты хранятся в:

```text
samples/presets
```

Через интерфейс можно:

- создать пресет;
- добавить строки пресета;
- использовать одну стратегию несколько раз в разные периоды;
- поставить «До конца» вместо даты окончания;
- сохранить пресет;
- перезаписать существующий пресет после подтверждения;
- удалить пресет.

При удалении пресета стратегии не удаляются.

## Ребалансировки

Ребалансировка — это когда одна стратегия участвует в одном пресете несколько раз, но с разными периодами и весами.

Так можно:

```text
strategy_a: 100%, 2024-01-01 00:00 — 2025-01-01 00:00
strategy_a: 10%,  2025-01-01 00:00 — до конца
```

Так нельзя:

```text
strategy_a: 100%, 2024-01-01 00:00 — 2025-01-01 00:00
strategy_a: 10%,  2024-06-01 00:00 — 2025-06-01 00:00
```

Потому что периоды одной и той же стратегии пересекаются, и непонятно, какой вес брать.

## Расчет

В блоке «Расчет» можно выбрать:

- стратегию;
- пресет.

Потом нужно указать период:

```text
Период с
Период по
```

Расчет всегда строится через `diff`.

Для пресета формула такая:

```text
preset_diff = сумма(strategy_diff × weight)
```

После этого из общего `diff` строятся:

- `accum`;
- `hwm`;
- `dd`;
- `mdd`.

## Метрики и ряды

### diff

Доходность за конкретный шаг.

### accum

Накопленная доходность за выбранный период.

На первой точке выбранного периода `accum = 0`.

### hwm

High Water Mark — максимум накопленной доходности на текущий момент.

### dd

Drawdown — текущая просадка в каждой точке.

### mdd

Max Drawdown series — худшая просадка, накопленная к каждой точке времени.

Пример:

```text
DD:
0
-0.02
-0.05
-0.01
-0.08
-0.03

MDD:
0
-0.02
-0.05
-0.05
-0.08
-0.08
```

То есть `DD` может восстанавливаться, а `MDD` показывает худшую уже случившуюся просадку.

## График

На графике есть тумблеры:

- `diff`;
- `accum`;
- `hwm`;
- `dd`;
- `mdd`.

Цвет рядом с тумблером совпадает с цветом линии на графике.

## API

### Получить стратегии

```http
GET /api/strategies
```

### Загрузить стратегию

```http
POST /api/strategies
```

Multipart form-data:

- `file` — CSV-файл;
- `name` — имя стратегии;
- `valueType` — `diff` или `accum`.

### Удалить стратегию

```http
DELETE /api/strategies/{file}
```

### Получить пресеты

```http
GET /api/presets
```

### Создать или перезаписать пресет

```http
POST /api/presets
```

JSON body:

```json
{
  "name": "preset_1",
  "overwrite": false,
  "items": [
    {
      "strategy": "strategy_a.csv",
      "weightPercent": 100,
      "date_from": "2024-01-01 00:00",
      "date_to": "2025-01-01 00:00",
      "untilEnd": false
    }
  ]
}
```

Если `untilEnd = true`, то `date_to` сохраняется как `null`.

### Удалить пресет

```http
DELETE /api/presets/{name}
```

### Запустить расчет

```http
POST /api/calculate
```

JSON body:

```json
{
  "targetType": "preset",
  "targetName": "preset_1",
  "periodFrom": "2024-01-01 00:00",
  "periodTo": "2025-01-01 00:00"
}
```

Для расчета одной стратегии:

```json
{
  "targetType": "strategy",
  "targetName": "strategy_a.csv",
  "periodFrom": "2024-01-01 00:00",
  "periodTo": "2025-01-01 00:00"
}
```

## Важные ограничения текущей версии

- Файлы стратегий ожидаются как две колонки: `timestamp,value`.
- Текущая загрузка принимает числовые значения как decimal-доходность. Значение `0.015` означает `1.5%`.
- Если загрузить `1.5` без знака `%`, это будет прочитано как `150%`, а не как `1.5%`.
- В дальнейшем нужно добавить явный выбор формата значений: decimal или percent.

## Обновление терминов: портфолио и торговые стратегии

С этого этапа CSV-файлы доходности называются **портфолио**, а слово **стратегии** используется для торговых правил поверх уже рассчитанного портфолио или пресета.

Текущая структура хранения:

```text
samples/portfolios   CSV портфолио в нормализованном формате timestamp,diff
samples/presets      JSON-пресеты из портфолио, весов и периодов
samples/strategies   JSON-конфиги торговых стратегий
samples/runs         JSON-результаты расчетов
```

В блоке “3. Расчет” пользователь выбирает портфолио или пресет и период. Блок “Стратегии” включается отдельным тумблером и применяет торговую стратегию к уже выбранному расчету.

Первая торговая стратегия — RSI:

- RSI считается по equity-кривой `1 + accum`;
- на графике доходность отображается как `accum`, то есть стартовая точка равна `0%`;
- параметры по умолчанию: период `14`, уровни `70/30`, базовая линия `50`;
- правило торговли long-only: покупка происходит при пересечении `buyLevel` сверху вниз (`previous RSI > buyLevel && current RSI <= buyLevel`), продажа — при пересечении `sellLevel` снизу вверх (`previous RSI < sellLevel && current RSI >= sellLevel`);
- первая точка периода стратегии не является сигнальной, а повторные покупки/продажи игнорируются, если позиция уже в нужном состоянии;
- сигнал исполняется со следующей точки, чтобы не было подглядывания в будущее;
- если период стратегии выходит за период расчета, показывается предупреждение, а отсутствующие данные заполняются по общему правилу.

После расчета стратегии UI показывает:

1. исходный график портфолио/пресета;
2. RSI-подграфик с линиями уровней;
3. отдельный итоговый график и таблицу доходности торговой стратегии.
