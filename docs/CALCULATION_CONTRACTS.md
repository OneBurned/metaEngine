# Calculation contracts and golden fixtures

Этот документ фиксирует расчетное поведение MetaEngine перед переносом
calculation engine из Node.js в C#. Машиночитаемые эталоны лежат в
`test/fixtures/golden` и должны использоваться обеими реализациями.

## Назначение

Golden fixture содержит небольшой входной ряд и ожидаемый результат. Тест
падает, если изменились формула, сигнал, момент исполнения, позиция, итоговый
`diff` или summary.

Golden tests не доказывают корректность любой возможной стратегии. Они защищают
уже согласованные ключевые сценарии от случайного изменения и дают общий эталон
для Node.js reference и C# production engine.

## Формат fixture

Каждый JSON содержит:

| Поле | Значение |
| --- | --- |
| `schemaVersion` | Версия структуры самого fixture |
| `contractVersion` | Версия расчетного контракта |
| `id` | Стабильный идентификатор сценария |
| `kind` | `base_calculation` или `trading_strategy` |
| `absoluteTolerance` | Допустимое абсолютное отклонение чисел |
| `input` | Timestamp, diff, таймфрейм и параметры |
| `expected` | Канонический diff, checkpoints и summary |

Timestamp хранится как Unix time в миллисекундах UTC. Доходности, веса и уровни
хранятся в decimal: `0.01` означает `1%`.

Пустой сигнал или исполнение хранится как `""`. Отсутствующее числовое значение,
например RSI до накопления периода или Local Accum вне ожидания TP, хранится как
`null`.

## Сравнение чисел

Версия контракта `1.0` использует IEEE 754 double и абсолютную погрешность
`1e-12`. Тест не округляет ожидаемые значения до процентов.

Frontend может округлять числа для отображения, но расчетный engine, API и
сохраненный `timestamp,diff` сравниваются с исходной точностью.

Если C# реализация требует другую числовую модель, изменение точности сначала
обсуждается как изменение расчетного контракта. Нельзя просто увеличить
tolerance, чтобы скрыть расхождение формул.

## Текущие эталоны

```text
base_metrics.json
  diff -> accum -> hwm -> dd -> mdd

rsi_strategy.json
  RSI по equity, buy/sell crossing и исполнение со следующей точки

mdd_mean_reversion_strategy.json
  уровни DD, целевые веса, восстановление, Local Accum и TP
```

Стратегические fixtures содержат полный выходной `resultDiffs`. Контрольные
точки дополнительно проверяют индикаторы, сигналы, исполнения, позицию и
производные метрики.

## Правила изменения

1. Исправление кода, не меняющее продуктовую формулу, не должно менять fixtures.
2. Намеренное изменение формулы сначала согласуется с пользователем.
3. После согласования повышается `contractVersion` и добавляется новый fixture
   или явно обновляется существующий с описанием причины.
4. Старые fixtures сохраняются, если production должен уметь воспроизводить
   историческую версию engine.
5. Новая мета-стратегия добавляет как минимум один golden fixture своего модуля.

Нельзя генерировать expected-часть автоматически во время теста тем же кодом,
который тестируется. Expected значения хранятся в Git и меняются только
осмысленным коммитом.

## Запуск

```bash
npm test
```

Файл `test/golden-contracts.test.js` загружает JSON, выполняет текущий Node.js
engine и сравнивает результат. Будущий C# test project должен читать те же JSON
без копирования ожидаемых значений в C# код.
