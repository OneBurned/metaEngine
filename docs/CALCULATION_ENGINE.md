# Production calculation engine

Этап P2b переносит базовые расчетные формулы MetaEngine из Node.js reference в
чистое доменное ядро C#. Оно не зависит от HTTP, PostgreSQL, Worker или
конкретной мета-стратегии и станет общим основанием для пресетов, RSI, MDD Mean
Reversion и будущих модулей.

## Вход и результат

Ядро получает канонический ряд `timestamp,diff`, период, исходный таймфрейм и
таймфрейм результата. Оно возвращает строки:

```text
timestamp,diff,accum,hwm,dd,mdd
```

и summary: начало, конец, количество точек, итоговый `accum`, итоговый HWM и
MDD. Формулы используют IEEE 754 double и должны совпадать с shared golden
fixture с абсолютным допуском `1e-12`.

Первая точка выбранного результата является baseline: `diff=0`, `accum=0`.
Начиная со второй точки:

```text
accum = (1 + diff) * (1 + previousAccum) - 1
hwm   = max(previousHwm, accum, 0)
dd    = (1 + accum) / (1 + hwm) - 1
mdd   = min(previousMdd, dd, 0)
```

## Период, пропуски и таймфрейм

Исходная сетка строится на всем выбранном периоде. Если timestamp отсутствует,
расчет использует `diff=0`, увеличивает полный счетчик пропусков и возвращает до
100 подробных warnings. Максимальный размер одной синхронно подготовленной
сетки ядра — 1 000 000 точек; resource quotas для jobs будут добавлены отдельно.

Исходные фиксированные таймфреймы: `1m`, `5m`, `15m`, `1h`, `1d`. Результат
можно оставить на исходном таймфрейме или укрупнить до `5m`, `15m`, `1h`, `1d`,
`1M`, `1Y`. Уменьшение таймфрейма запрещено. Месячные и годовые границы
календарные и считаются в UTC. Период должен быть выровнен так, чтобы границы
результата существовали на исходной сетке.

## Граница -100%

`diff < -1` отклоняется при production-импорте и повторно проверяется ядром.
`diff = -1` означает полную потерю капитала. После нее equity остается нулевой,
даже если следующие точки положительные. При укрупнении таймфрейма последующие
периоды получают `diff=0`, а не `NaN` от деления нуля на ноль.

## Проверки

`MetaEngine.DomainTests` читает тот же `test/fixtures/golden/base_metrics.json`,
что и Node.js suite. Дополнительные тесты проверяют пропуски, направление
таймфрейма, календарные границы, ограничение warnings и полную потерю капитала.

```bash
dotnet test tests/MetaEngine.DomainTests/MetaEngine.DomainTests.csproj
npm test
```
